Multi-Agent Pipeline

03 / 08
Agent Model MCP Role Responsibility Key Output
OrchestratorAgent gpt-4o Supervisor Intent classification · parallel dispatch · Tree-of-Thought for complex queries · response aggregation Final SSE answer streamed to UI
SearchAgent gpt-5.4 Active Caller Extracts search params via LLM · calls flight / hotel / car MCP tools · formats structured results Flight cards · hotel list · car options
PolicyAgent gpt-4o Reader Deterministic rule engine — checks class limits, advance booking windows, per-diem caps, preferred airlines PolicyVerdict (approved / warn / block)
DestinationAgent gpt-5.4 Reader Enriches responses with visa requirements, weather, local tips, currency, timezone Structured destination briefing JSON
BookingAgent gpt-5.4 Active Caller Creates / cancels bookings · generates NVA-YYYY-NNNNN references · writes to nva_bookings Booking confirmation + reference ID
JudgeAgent gpt-4o / Claude Opus No MCP LLM-as-judge — scores every response on 5 criteria async; fires after SSE stream closes Eval score doc in nva_eval_scores

Advanced reasoning — Tree of Thought

Complex queries (multi-city, open-ended policy questions) trigger ToT strategy: the Orchestrator generates 3 independent reasoning branches in parallel, scores each with the LLM, and selects the highest-scoring branch before streaming the final answer. ToT SSE events (tot_start / tot_branch / tot_evaluate / tot_selected) are visible in the MCP Inspector panel.

Three-Sidecar MCP Architecture

04 / 08

✈ Amadeus Travel MCP port 8101

  • Tool: search_flights
  • 36,288 static flight records
  • 64 routes · 30 cities worldwide
  • Live Amadeus API fallback (sandbox)
  • Cabin class / date / price filters

🚗 Car Rental MCP port 8102

  • Tools: search_cars · book_car · cancel_car
  • 6 agencies · 6 vehicle classes
  • 20 IATA city codes
  • Deterministic rate seeding (md5 hash)
  • Confirmation: CR-YYYYMMDD-XXXX

🏨 Hotel Booking MCP port 8103

  • Tools: search_hotels · book_hotel · cancel_hotel
  • 12 hotel brands · 4 star tiers
  • 20 IATA city codes
  • Tier multipliers: 1× → 5.5×
  • Confirmation: HB-YYYYMMDD-XXXX

Protocol

Each sidecar exposes three endpoints:

/mcp — FastMCP Streamable-HTTP protocol (agent calls)

/health — JSON health + version + data stats

/tools — full inputSchema listing (Admin panel probe)

SearchAgent routes each tool to the correct sidecar at runtime: flights → 8101, hotels → 8103, cars → 8102.

Admin visibility

Admin → MCP Bindings tab probes all three sidecars on load:

● Green status dot + latency badge (e.g. 18ms) for each

● Tool schemas with expand/collapse inputSchema viewer

● Per-agent binding matrix — which agents call which tools

● Every tool call emits mcp_tool_call / mcp_tool_result SSE events visible in the MCP Inspector panel

3-Step Booking Wizard

06 / 08
①

Passenger Details

First/last name · date of birth · passport number + expiry · nationality · contact email. Required fields validated before Next is enabled.

②

Seat Preference

Window / Aisle / No preference radio group. Meal preference: Standard · Vegetarian · Halal · Kosher. Optional special assistance field.

③

Review & Confirm

Full summary card with flight, passenger, seat, total. Policy compliance checkbox required before Confirm Booking is enabled.

✓

Confirmation

NVA-2026-NNNNN reference displayed. Download Itinerary exports a print-ready HTML file. Reference echoed into the chat thread.

Backend endpoint

POST /api/chat/bookings

→ generates NVA-YYYY-NNNNN

→ persists to nva_bookings collection

→ returns {booking_id, reference, status, created_at}


GET /api/chat/bookings/{id}

→ user-scoped booking detail retrieval

UX design decisions

  • Modal backdrop click cancels at any pre-confirmation step — no partial booking created
  • Stepper header shows numbered circles (active = navy, done = green ✓, connecting line turns green)
  • Confirm button disabled until policy checkbox is checked and booking is not submitting
  • Error banner surfaced for failed API call with retry path
  • Itinerary download is client-side only — Blob → anchor click, no server roundtrip

Admin Console — 12 Tabs

07 / 08

⚙ Model Selection

Switch any agent between Azure OpenAI · OpenAI · Anthropic · Ollama. Temperature + max_tokens sliders. Changes hot-reload via agents.json — no restart required.

✏ Agent Prompts

Edit system prompt for each agent in a monospace textarea. ● dot on tab = unsaved changes. Save & Reload writes to config/prompts.json; next request uses the new prompt. Reset to Default available.

🔌 MCP Bindings

Live status dots + latency (ms) for all 3 MCP sidecars. Per-agent tool binding matrix. Full inputSchema viewer with expand/collapse per tool.

📋 Policy

View and edit all 6 corporate travel policies: Standard Domestic · International · Executive · Company Executive · Sales Executive · Conference Traveller. Class limits, per-diem caps, advance booking windows.

👤 Employee Data

100 employee records with department, travel tier, and policy assignment. CSV upload. Used by PolicyAgent to resolve per-employee limits.

🔔 Notifications

Real-time admin alerts: new bookings, policy violations flagged by PolicyAgent, JudgeAgent low-score warnings. Dismissible notification stack.

📒 Audit Log

Searchable agent action log. Auto-refresh every 10s. Shows timestamp, user, agent, action, details. CSV export. Persists to nva_audit_log.

💰 Billing

Token usage and estimated cost by model / day. Pure CSS bar chart. Auto-refresh every 60s. CSV export. Reads from nva_billing.

📊 Eval Metrics

JudgeAgent 5-criteria scores (accuracy, policy, helpfulness, tone, safety). Pass/fail rate. Auto-refresh every 15s. Score drill-down per conversation. JSON export.

📡 Observability

Per-conversation agent latency Gantt trace. Shows wall-clock time for each sub-agent call and MCP tool invocation. Identify bottlenecks at a glance.

💬 Chat History

Searchable conversation list with starred messages, reaction badges, eval score indicators, and inline thread expansion. Export conversation as JSON.

🗂 Mock Data

Browse all 88k flight, hotel, and car records used by the MCP sidecars. Filter by origin/city/date. Useful for demoing specific search scenarios.

LLM Observability & Tech Stack

08 / 08

Per-message observability layers

🔍 Debug Mode Panel

Toggle with 🐞 button (persisted in localStorage). Collapsible panel under each assistant message listing: agents invoked, MCP tool name + input/output JSON, token counts, model used, latency. Powered by metadata on each SSE done event.

📈 Performance Metrics Row

Always visible below every assistant bubble: latency chip (ms, computed frontend-side), input/output token counts, model label. Not gated — visible to all users.

👍👎🚩 Reaction Bar

Thumbs up / thumbs down / flag per message. Optional comment modal. Written to nva_feedback via POST /api/chat/feedback. Reactions visible in Admin → Chat History tab with gold border on starred messages.
